# Usage and credits

The MOSTLY AI Platform at https://app.mostly.ai provides a daily limit of 5 credits to train generators, generate synthetic datasets, and use the Assistant. Enterprise deployments have a usage-based configuration which can be adjusted according to the organization's needs.

<CustomCallout>
**Note**
When you use up your credits, you can no longer train new generators, create new synthetic datasets, or start Assistant chats.

For more credits on https://app.mostly.ai or for the configuration of credits on Enterprise deployments, contact MOSTLY AI.
</CustomCallout>

## Usage

Usage is tracked based on consumed **Total CPU time** and **Total GPU time**.

For each generator and synthetic dataset, you will find the total virtual CPU/GPU time consumed in the **Training status** section.

**Generator details: Compute consumption**
GPU and CPU time are measured in seconds. The total time is the sum of the GPU and CPU time used. GPU time is only available when a GPU was selected to train a generator.

## Credits

Credits are consumed based on the total virtual CPU and GPU time used to train generators and generate synthetic datasets.

You can use the Python SDK `mostly.me()` primitive to check the usage on your account.
```python filename="python" copy
from mostlyai.sdk import MostlyAI
mostly = MostlyAI(api_key="INSERT_API_KEY")
mostly.me()
```
The result is a Python dictionary and you can find details about your current credits usage and limit in the `usage` key.

# Export and import generators

MOSTLY AI allows you to export already trained generators from one environment and import them into another.

The export and import of generators is useful when you need to have a separation of concerns. For example, you can train a generator with the Python SDK in an environment where you have direct and secure access to the original data, and where it is safe to train generators on the original data. In this scenario, the data remains private as the generator learns only the statistical properties and never memorizes specific data points.

You can then export the trained generator from the secure environment and import it into a running instance of the MOSTLY AI Platform. That instance can be made available to everyone in your organization. People can then use the Platform to generate privacy-safe synthetic data and extract data insights with the [Assistant](/assistant) without access to the original data.

## Export a generator

<Tabs items={['UI', 'Python SDK']}>
<Tabs.Tab>

If you use the UI of the MOSTLY AI Platform, you can export a generator in one of two ways.

**Prerequisites**

To export a generator, you must be its creator or have the Admin role in the organization.

**Steps**

* Export a generator directly from the **Generators** page.
    1. From the **Generators** page, click the kebab menu of a generator, and select **Export to file**.
        <Image 
            src="/docs/docimages/generators/export-import/export-02-generators-page-kebab-menu-select-export-to-file.webp" 
            alt="MOSTLY AI - Export a generator - 01 - From Generators page, kebab menu, Export to file" 
            width={800} 
            height={150}
        />
* Export a generator after you open it.
    1. From the **Generators** page, click a generator to open it.
    2. Click the kebab menu in the upper right, and select **Export to file**.
        <Image 
            src="/docs/docimages/generators/export-import/export-01-kebab-menu-select-export-to-file.webp" 
            alt="MOSTLY AI - Export a generator - 01 - Open kebab menu and select Export to file" 
            width={800} 
            height={150}
        />
</Tabs.Tab>
<Tabs.Tab>

<CustomCallout type="API">
Available in both [Local and Client modes](/python-sdk#local-and-client-modes).
</CustomCallout>

With the Python SDK, you can export a generator to a ZIP archive with the `export_to_file()` method. Optionally, you can specify a filename.

```python filename="python" copy
g = mostly.generators.get('INSERT_GENERATOR_ID')

# Exports with a default filename 'generator-UUID-time-high-octet.zip' 
g.export_to_file() 

# Export with a specified filename
g.export_to_file('INSERT_GENERATOR_FILENAME.zip')
```
</Tabs.Tab>

</Tabs>

**Result**

The generator is exported as a `ZIP` archive. You can now import it into another instance of the MOSTLY AI Platform (local or remote).

## Import a generator

Learn how to import generators into another instance of the MOSTLY AI Platform with the UI and the Python SDK. You can do so in a local environment as well as in a remotely running MOSTLY AI Platform.

<Tabs items={['UI', 'Python SDK']}>
<Tabs.Tab>
You can use the UI of another instance of the MOSTLY AI Platform to import exported generators.

**Steps**

1. On the **Generators** page, click **Import from file**.
    <Image 
        src="/docs/docimages/generators/export-import/import-01-click-import-from-file.webp" 
        alt="MOSTLY AI - Import a generator - 01 - Click Import from file" 
        width={800} 
        height={150}
    />
2. On the **Import Generator** prompt, click in the upload area to browse or drag-and-drop the exported generator ZIP file.
    <Image 
        src="/docs/docimages/generators/export-import/import-02-click-upload-area.webp" 
        alt="MOSTLY AI - Import a generator - 02 - Click upload area" 
        width={800} 
        height={150}
    />
3. Click **Proceed**.

**Result**

The imported generator opens immediately. You can now use it to [generate synthetic data](/synthetic-datasets/generate) in the new environment.
</Tabs.Tab>
<Tabs.Tab>

<CustomCallout type="API">
Available in both [Local and Client modes](/python-sdk#local-and-client-modes).
</CustomCallout>

You can import a generator with the `import_from_file()` method.

```python filename="python" copy
g = mostly.generators.import_from_file('path/to/generator.zip')
```
</Tabs.Tab>
</Tabs>

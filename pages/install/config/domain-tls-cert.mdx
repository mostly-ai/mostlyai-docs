# Configure your domain TLS certificate

To deploy MOSTLY AI, you need to configure TLS certificates for your fully-qualified domain name (FQDN) to enable secure access to the MOSTLY AI web app.

**Prerequisites**

* Obtain your FQDN TLS certificate from your IT department.
* If you have access to the Kubernetes cluster and the certificate, you can create the secret in your Kubernetes cluster as follows:
    ```shell copy filename="shell" /INSERT_MOSTLY_TLS_SECRET_NAME/
    kubectl create secret tls INSERT_MOSTLY_TLS_SECRET_NAME --cert=cert.crt --key=cert.key -n <namespace>
    ```

**Steps**

1. Edit the `values.yaml` file.
2. Set the TLS secret name to the name of the secret you created under `_customerInstallation.deploymentSettings.tlsSecretName`.
    ```yaml filename="values.yaml" {4} /INSERT_MOSTLY_TLS_SECRET_NAME/
        _customerInstallation:
          ...
          deploymentSettings:
            tlsSecretName: &tlsSecretName INSERT_MOSTLY_TLS_SECRET_NAME
          ...
    ```
3. Save the file.

**What's next**

You can continue with the [deployment](../deploy) of MOSTLY AI.

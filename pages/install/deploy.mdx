# Deploy MOSTLY AI

A system administrator of a Kubernetes cluster can best navigate the deployment of MOSTLY AI. The deployment requires the management and mapping of permissions for the storage, nodes, ingress, and secrets to internal repositories, as well as the definition of requests for the pods. The values.yaml file includes suggestions on minimum requirements and guidance on where to map each of those parameters.

## Deployment targets

You can deploy MOSTLY AI in an on-premises Kubernetes cluster or in managed Kubernetes services.

<Callout type='info'>
Contact MOSTLY AI if you need help deploying to a specific managed Kubernetes service.
</Callout>

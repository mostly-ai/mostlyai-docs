# Use a SQL Server database for synthetic data

MOSTLY AI can use a Microsoft SQL Server database as a source for original data as well as a destination to deliver synthetic data. To do so, you need to create SQL Server connectors.

For each SQL Server data source or destination, you need a separate connector.

**Prerequisites**

Obtain the SQL Server connection details.

* host
* port
* credentials
* database name

<Tabs items={['UI', 'Python SDK - Local mode', 'Python SDK - Client mode']}>
<Tabs.Tab>
If you use the web application, create a new SQL Server connector from the **Connectors** page.

**Steps**

1. From the **Connectors** tab, click **Create connector**.
    <Image
        src="/docs/docimages/connectors/create-connector-01-click.webp"
        alt="Click Create connector button"
        width={800}
        height={500}
    />
2. From the **Create a new connector** window, select **Microsoft SQL Server**.
    <Image
        src="/docs/docimages/connectors/sql-server/create-connector-02-select-sql-server.webp"
        alt="Select SQL Server connector"
        width={800}
        height={500}
    />
3. From the **New connector** window, configure the connector.
    1. For **Name**, enter a name that you can distinguish from other connectors.
    2. For **Access type**, select whether you want to use the connector as a **source** or **destination**.
    3. For **Host**, enter the SQL Server database hostname.
    4. For **Port**, enter the database port.
        <Callout type="info">
        The default port for SQL Server databases is 1433.
        </Callout>
    5. For **Username** and **Password**, enter your SQL Server database credentials.
    6. For **Database**, enter the name of the database.
        <Image
            src="/docs/docimages/connectors/sql-server/create-connector-03-configure-sql-server.webp" alt="Configure SQL Server connector"
            width={400}
            height={500}
        />
5. Click **Save** to save your new SQL Server connector.<br /><br />
    MOSTLY AI tests the connection. If you see an error, check the connection details, update them, and click **Save** again.<br /><br />
    You can click **Save anyway** to save the connector disregarding any errors.
</Tabs.Tab>
<Tabs.Tab>
If you use the Python SDK in Local mode, you first need to install the Microsoft SQL Server drivers for your operating system.

1. Install the ODBC driver for your operating system.
    * [Linux](https://learn.microsoft.com/en-us/sql/connect/odbc/linux-mac/installing-the-microsoft-odbc-driver-for-sql-server)
    * [macOS](https://learn.microsoft.com/en-us/sql/connect/odbc/linux-mac/install-microsoft-odbc-driver-sql-server-macos)
    * [Windows](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server?view=sql-server-ver16)
2. Create a SQL Server connector with `mostly.connect()`. Define a connector configuration dictionary as shown below.
```python copy filename="python"
from mostlyai.sdk import MostlyAI

mostly = MostlyAI(local=True)

c = mostly.connect({
    "name": "MS SQL Server Connector",
    "type": "MSSQL",
    "access_type": "SOURCE",
    "config": {
        "host": "mssql.hostname.com",
        "port": "1433",
        "username": "admin",
    },
    "secrets": {
        "password": "******"
    },
    "test_connection": True
})
```

The local server tests the connection to the MS SQL database, and if successful, the connector is created. Bear in mind, this tests the connection from the local environment to the server.

If you see an error, check the connection details and try again.
</Tabs.Tab>
<Tabs.Tab>
If you use the Python SDK in Client mode to connect to a remote Platform, connect to the remote platform via `MostlyAI()` with `base_url` and `api_key`, and then create a SQL Server connector.

```python copy filename="python"
from mostlyai.sdk import MostlyAI

mostly_remote = MostlyAI(base_url="https://api.mostly.ai", api_key="INSERT_API_KEY")

c = mostly_remote.connect({
    "name": "MS SQL Server Connector",
    "type": "MSSQL",
    "access_type": "SOURCE",
    "config": {
        "host": "mssql.hostname.com",
        "port": "1433",
        "username": "admin",
    },
    "secrets": {
        "password": "******"
    },
    "test_connection": True
})
```

The platform tests the connection to the MS SQL database, and if successful, the connector is created. Bear in mind, this tests the connection from the remote Platform to the server.

If you see an error, check the connection details and try again.
</Tabs.Tab>
</Tabs>

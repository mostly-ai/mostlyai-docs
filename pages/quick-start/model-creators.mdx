# Quick start for Model creators

As Model creators, you carry the torch of democratizing data within and outside of your organization!

Here is how you can quickly train a new generator on original tabular data. Your peers, colleagues, and third parties can then use the generator to create privacy-safe synthetic data.

<Steps>
## Step 1: Train a generator
<Tabs items={['UI', 'Python SDK']}>
<Tabs.Tab>
1. In MOSTLY AI, select the **Generators** page.
2. Click **+ New generator**.
    <Image 
        src="/docs/docimages/quickstart/model-creators/train-generator-01-click-new-generator.webp" 
        alt="MOSTLY AI - Train a generator - 01 - Click New generator" 
        width={800} 
        height={150}
    />
3. Select **Upload file**, browse for a dataset, and click **Proceed**.
    <CustomCallout>
    **Note**<br />
    If you don't have one, download the [UCI Adult income dataset](/datasets/us-census-income.csv.gz).
    </CustomCallout>
    <Image 
        src="/docs/docimages/quickstart/model-creators/train-generator-02-click-upload-files.webp" 
        alt="MOSTLY AI - Train a generator - 02 - Click Upload files" 
        width={800} 
        height={150}
    />
4. At the **Data configuration** step, click **Configure models**.
    <Image 
        src="/docs/docimages/quickstart/model-creators/train-generator-03-click-configure-models.webp" 
        alt="MOSTLY AI - Train a generator - 04 - Click Configure models" 
        width={800} 
        height={150}
    />
5. At the **Model configuration** step, click **Start training**.
    <Image 
        src="/docs/docimages/quickstart/model-creators/train-generator-04-click-start-training.webp" 
        alt="MOSTLY AI - Train a generator - 04 - Click Start training" 
        width={800} 
        height={150}
    />
</Tabs.Tab>
<Tabs.Tab>
1. [Install the MOSTLY AI Python SDK](/python-sdk#installation).
2. Create your first generator using the [US Census Income](/datasets#us-census-income-dataset) dataset, start its training, and wait for it to finish.
    <CustomCallout>
    Generator training uses compute from the local environment.
    </CustomCallout>
    <Tabs items={['Local mode', 'Client mode']}>
    <Tabs.Tab>
    ```python copy filename="python" {7}
    # 1. Load original data into a pd.DataFrame
    import pandas as pd
    df = pd.read_csv("https://mostly.ai/docs/datasets/us-census-income.csv.gz")

    # 2. Instantiate in Local use
    from mostlyai.sdk import MostlyAI
    mostly = MostlyAI(local=True)
    
    # 3. Create a generator and launch its training
    g = mostly.train(data=df, start=True, wait=True)
    ```
    </Tabs.Tab>
    <Tabs.Tab>
    ```python copy filename="python" {7}
    # 1. Load original data into a pd.DataFrame
    import pandas as pd
    df = pd.read_csv("https://mostly.ai/docs/datasets/us-census-income.csv.gz")

    # 2. Instantiate in Client mode by connecting to a remote Platform instance
    from mostlyai.sdk import MostlyAI
    mostly_remote = MostlyAI(base_url="https://app.mostly.ai", api_key="INSERT_API_KEY")
    
    # 3. Create a generator and launch its training
    g = mostly.train(data=df, start=True, wait=True)
    ```
    </Tabs.Tab>
    </Tabs>
</Tabs.Tab>
<Tabs.Tab>
1. Install the MOSTLY AI Synthetic Data SDK.<br />
    ```python copy filename="bash"
    pip install -U mostlyai
    ```
2. Get an [API key](/api-key).
3. Create your first generator using the [US Census Income](/datasets#us-census-income-dataset) dataset, start its training, and wait for it finish.
    <CustomCallout>
    Generator training uses compute from the remote MOSTLY AI Platform.
    </CustomCallout>
    ```python copy filename="python" {7-10}
    # 1. Load original data into a pd.DataFrame
    import pandas as pd
    df = pd.read_csv("https://mostly.ai/docs/datasets/us-census-income.csv.gz")

    # 2. Connect the MOSTLY AI SDK to a remote Platform
    from mostlyai.sdk import MostlyAI
    mostly_remote = MostlyAI(
        base_url="https://app.mostly.ai",
        api_key="INSERT_API_KEY"
    )
    
    # 3. Create a generator and launch its training
    g = mostly_remote.train(data=df, start=True, wait=True)
    ```
Once training has finished, you, your peers, colleagues, or any third party can generate synthetic data.
</Tabs.Tab>
</Tabs>

**What's next**

After the generator training completes, you can generate synthetic data. You can also export it and share it with Data consumers so they can too generate synthetic data.

## Step 2: Export and share the generator
<Tabs items={['UI', 'Python SDK']}>
<Tabs.Tab>
1. In MOSTLY AI, select the **Generators** page.
2. Open the generator.
3. Click the kebab menu and select **Export generator**.
    <Image
        src="/docs/docimages/quickstart/model-creators/export-share-01-kebab-select-export-generator.webp"
        alt="MOSTLY AI - Export and share the generator - 01 - Select Export generator"
        width={800}
        height={150}
    />
4. Save the generator locally.

**Result**

The generator is exported as a `ZIP` archive.

</Tabs.Tab>
<Tabs.Tab>

<CustomCallout type="API">
Available in both [Local and Client modes](/python-sdk#local-and-client-modes).
</CustomCallout>

On a generator object, use the `export_to_file()` method to export a generator to a `ZIP` archive.
    ```python copy filename="python"
    g.export_to_file()
    ```
    The generator is exported and saved as a `ZIP` archive.
</Tabs.Tab>
</Tabs>
**What's next**

You can now share the `ZIP` archive with Data consumers (peers, colleagues, or third parties) so they can import the generator in a MOSTLY AI Platform instance to generate synthetic data.
</Steps>
To generate synthetic data, see [Quickstart for Data consumers](data-consumers).
